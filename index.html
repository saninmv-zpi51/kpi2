Тема власного WEB-застосунку

Приклад (замінити на свою предметну галузь): WEB-застосунок для обліку тренувань, харчування та прогресу користувача.

Мета створення WEB-застосунку

Автоматизувати ведення щоденника тренувань і харчування, забезпечити наочну статистику та контроль цілей.

Скорочений опис предметного середовища (бізнес-логіка)

Користувач додає записи про тренування, харчування та вагу. Система зберігає дані, показує історію, підраховує підсумки та формує звіти за період.

Функціональні вимоги

  • Додавання/редагування/видалення записів (тренування, харчування, вага).
  • Перегляд списку записів за датами.
  • Фільтрація/пошук.
  • Виведення підсумків за день/тиждень.

Нефункціональні вимоги

  • Зручний інтерфейс, адаптивність під різні екрани.
  • Швидке завантаження сторінок.
  • Коректна семантична розмітка HTML.
  • Кросбраузерність (Chrome/Firefox/Edge).

Структура головної сторінки (семантична верстка)

<header>...</header>
              <nav>...</nav>
              <main>
              <section>...</section>
              <section>...</section>
              </main>
              <footer>...</footer>

У цій лабораторній також окремо наводяться фрагменти HTML-коду для таблиці, зображення, списків і форми (та скріншоти результату).

HTML-код таблиці

<table>
              <caption>Приклад таблиці</caption>
                <thead>
               <tr><th>Колонка 1</th><th>Колонка 2</th></tr>
                </thead>
              <tbody>
                <tr><td>Дані 1</td><td>Дані 2</td></tr>
              </tbody>
              </table>

Теорія (коротко)

  • <table> — контейнер таблиці.
  • <tr> — рядок; <th> — заголовкова комірка; <td> — звичайна комірка.
  • <caption> — підпис таблиці; <thead>/<tbody> — логічні частини.

Скріншот таблиці вставиш тут окремо як <img> після того, як зробиш скрін.

HTML-код зображення

<img src="images/example.png" alt="Опис зображення" width="400">

Теорія (атрибути)

  • src — шлях до зображення.
  • alt — альтернативний текст (важливо для доступності та якщо файл не завантажився).
  • width/height — розміри (краще контролювати через CSS, але в лабі можна показати атрибути).

HTML-код списків

<ul>
  <li>Маркірований пункт 1</li>
  <li>Маркірований пункт 2</li>
</ul>

<ol>
  <li>Нумерований пункт 1</li>
  <li>Нумерований пункт 2</li>
</ol>

Теорія

  • <ul> — маркірований список, <ol> — нумерований, <li> — елемент списку.
  • Атрибут type для <ol> може змінювати нумерацію (1, A, a, I, i).

HTML-код форми

<form action="#" method="post">
  <label>Ім'я:
    <input type="text" name="name" required>
  </label>

  <label>Email:
    <input type="email" name="email" required>
  </label>

  <label>Повідомлення:
    <textarea name="message" rows="4"></textarea>
  </label>

  <button type="submit">Надіслати</button>
</form>
